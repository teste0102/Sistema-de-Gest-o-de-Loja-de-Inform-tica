📋 PASSO 1: PREPARAR SEU COMPUTADOR
═══════════════════════════════════════════════════════════════════════════

Você precisa ter instalado (se não tiver, baixe e instale):

1️⃣ PYTHON 3.11+
   Acesse: https://www.python.org/downloads/
   Clique em "Downloads" → selecione Python 3.11 ou superior
   ⚠️  IMPORTANTE: Marque "Add Python to PATH" durante instalação
   
   Verificar: Abra PowerShell e digite:
   python --version

2️⃣ NODE.JS 18+
   Acesse: https://nodejs.org/
   Baixe a versão LTS
   Execute o instalador
   
   Verificar: Abra PowerShell e digite:
   node --version
   npm --version

3️⃣ GIT
   Acesse: https://git-scm.com/download/win
   Instale com as opções padrão
   
   Verificar: Abra PowerShell e digite:
   git --version

4️⃣ DOCKER DESKTOP (Opcional, mas Recomendado)
   Acesse: https://www.docker.com/products/docker-desktop
   Instale
   Inicie o Docker Desktop

═══════════════════════════════════════════════════════════════════════════

📥 PASSO 2: CLONAR O REPOSITÓRIO
═══════════════════════════════════════════════════════════════════════════

1. Abra PowerShell (pesquise "PowerShell" no Windows)

2. Digite os comandos abaixo (um por um):

   git clone https://github.com/teste0102/Sistema-de-Gest-o-de-Loja-de-Inform-tica.git

   cd Sistema-de-Gest-o-de-Loja-de-Inform-tica

   git checkout claude/loja-informatica-setup-6rgvdu

═══════════════════════════════════════════════════════════════════════════

🔧 PASSO 3: INSTALAÇÃO AUTOMÁTICA
═══════════════════════════════════════════════════════════════════════════

Você está na pasta "Sistema-de-Gest-o-de-Loja-de-Inform-tica"

🎯 OPÇÃO A: Clique duas vezes no arquivo "install-windows.bat"

OU

🎯 OPÇÃO B: No PowerShell, digite:

   .\install-windows.bat

Aguarde a instalação completar. Você verá mensagens de progresso.

═══════════════════════════════════════════════════════════════════════════

✨ PASSO 4: INICIAR A APLICAÇÃO
═══════════════════════════════════════════════════════════════════════════

Agora você precisa abrir 3 PowerShells DIFERENTES:

┌─────────────────────────────────────────────────────────────────────────┐
│ TERMINAL 1️⃣ - BANCO DE DADOS                                            │
└─────────────────────────────────────────────────────────────────────────┘

Abra um novo PowerShell e execute:

   docker-compose up -d postgres pgadmin

✓ Verá mensagens de início
✓ PostgreSQL estará pronto em: localhost:5432
✓ pgAdmin em: http://localhost:5050

⚠️  Se Docker não estiver instalado, você pode:
- Instalar Docker Desktop: https://www.docker.com/products/docker-desktop
- OU usar PostgreSQL local (mais complexo)

┌─────────────────────────────────────────────────────────────────────────┐
│ TERMINAL 2️⃣ - BACKEND (Python/FastAPI)                                 │
└─────────────────────────────────────────────────────────────────────────┘

Abra um novo PowerShell e execute:

   cd backend
   venv\Scripts\activate
   python main.py

✓ Verá a mensagem "Uvicorn running on http://127.0.0.1:8000"
✓ Backend estará pronto em: http://localhost:8000
✓ API interativa em: http://localhost:8000/docs

┌─────────────────────────────────────────────────────────────────────────┐
│ TERMINAL 3️⃣ - FRONTEND (React)                                         │
└─────────────────────────────────────────────────────────────────────────┘

Abra um novo PowerShell e execute:

   cd frontend
   npm start

✓ O navegador abrirá automaticamente
✓ Frontend estará em: http://localhost:3000

═══════════════════════════════════════════════════════════════════════════

🐛 PROBLEMAS?
═══════════════════════════════════════════════════════════════════════════

❌ ERRO: "python: command not found"
   ✓ Python não está no PATH
   ✓ Reinstale Python e MARQUE "Add Python to PATH"
   ✓ Reinicie o PowerShell

❌ ERRO: "node: command not found"
   ✓ Node.js não está no PATH
   ✓ Reinstale Node.js
   ✓ Reinicie o PowerShell

❌ ERRO: "Connection refused: localhost:5432"
   ✓ Docker não está rodando
   ✓ Abra Docker Desktop
   ✓ Execute: docker-compose up postgres

❌ ERRO: "Porta 3000 já em uso"
   ✓ Outro programa está usando a porta
   ✓ No Terminal 3, digite: PORT=3001 npm start

❌ ERRO: ".env file not found"
   ✓ O script deveria ter criado
   ✓ Copie manualmente: copy backend\.env.example backend\.env
